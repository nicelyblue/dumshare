# Security Policy

## Supported Versions

Security fixes are applied to the latest main branch and current release tag.

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Report suspected vulnerabilities privately to the maintainers with:

- description of the issue
- reproduction steps
- potential impact
- affected versions/commits

You should receive an acknowledgement within 5 business days.

## Sensitive Data Rules

- Never commit API keys, tokens, credentials, signing files, or keystores.
- Never commit `.env` with real values.
- Do not publish crash dumps or screenshots containing personal data.
