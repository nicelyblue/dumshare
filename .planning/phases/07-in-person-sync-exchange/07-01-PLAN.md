## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| contributor QR payload -> organizer parser | Untrusted data enters trusted session bootstrap logic |
| session establishment caller -> organizer-only operation | Unauthorized actor may attempt to start sync session |
| checkpoint reads/writes -> delta event query | Incorrect checkpoint handling may leak or skip events |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | T | `src/domain/sync/qr-request.ts` | mitigate | Strictly validate decoded QR fields and reject malformed/negative checkpoint values with deterministic errors. |
| T-07-02 | E | `src/domain/sync/session.ts` | mitigate | Call `assertOrganizerSyncHub` before accepting sync request or emitting session descriptor. |
| T-07-03 | I | `src/domain/events/repository.ts` | mitigate | Query deltas by `sequence > checkpoint`, preserve ascending order, and persist explicit peer checkpoints. |
| T-07-04 | R | `src/domain/events/repository.ts` checkpoint updates | mitigate | Keep checkpoint persistence explicit per `peer_id` to support deterministic audit and replay of sync boundaries. |
</threat_model>
