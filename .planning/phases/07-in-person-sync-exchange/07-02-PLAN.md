<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| remote sync events -> local append path | Untrusted remote events may include duplicates or tampered identifiers |
| checkpoint values -> delta selection | Incorrect checkpoints can cause missing or duplicate event transfers |
| exchange status output -> UI | Ambiguous/non-plain status text can hide transfer state from users |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-05 | T | `src/domain/sync/exchange.ts` remote apply path | mitigate | Validate and dedupe incoming events by `id` before append; preserve deterministic order. |
| T-07-06 | D | `src/domain/sync/exchange.ts` checkpoint update logic | mitigate | Advance checkpoint only after successful exchange calculation, using max observed sequence to avoid replay storms. |
| T-07-07 | I | status timeline generation | mitigate | Emit explicit plain-language statuses with send/receive counts to prevent hidden transfer ambiguity. |
| T-07-08 | R | bidirectional exchange result | mitigate | Return explicit upload/download event IDs and counts to support auditability of one-session sync outcome. |
</threat_model>
