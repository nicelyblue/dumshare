## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| contributor submission payload -> replay parser | Untrusted client-controlled proposal data enters deterministic state machine |
| review event actor -> approval authority gate | Potential spoofed reviewer identity attempts privileged decision path |
| pending queue -> approved entries | Incorrect transition can corrupt approved ledger state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | T | `replay.ts` submission payload parser | mitigate | Validate required fields, enum values, and payload shape before queue mutation. |
| T-06-02 | E | `replay.ts` review branch | mitigate | Enforce `assertOrganizerApprovalAuthority` for every review event before state transition. |
| T-06-03 | R | `replay.ts` reviewed decision trail | mitigate | Persist immutable review metadata (`submissionId`, reviewer, timestamp, sourceEventId`) for auditability. |
| T-06-04 | I | approved-entry transition logic | mitigate | Separate approved vs rejected branches and reject unknown/duplicate review transitions. |
</threat_model>
